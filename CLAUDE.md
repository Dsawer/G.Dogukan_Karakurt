# CV Site — Gürkan Doğukan Karakurt

Personal academic website for METU graduate studies HW1. Next.js 15 (App Router) +
Tailwind v4 + TypeScript, deployed to Vercel. English-only.

## Project layout

```

  layout.tsx                Root layout, fonts, metadata, theme provider, nav + footer
  page.tsx                  Homepage composing all sections
  globals.css               Tailwind v4 + CSS-variable theme (light/dark)

  cv/
    layout.tsx              Escape the root layout for the print route
```

## Editing content

`lib/content.ts` is the only file you need to touch to update copy. Both the
homepage and the printable `/cv` route consume the same exported objects:

- `profile` — name, headline, roles, about paragraphs, research interests, links
- `iconLinks` — ORCID / Scholar / LinkedIn (no GitHub, no email, no phone)
- `education` / `experience` — timeline entries
- `featuredProject` — Diyet Cebimde card content
- `skills` — four grouped skill lists
- `navItems` — section anchors used by the nav and scroll-spy

After editing content, regenerate the PDF: `npm run pdf`.

## Stack conventions

- Server components by default. Add `"use client"` only when a component needs
  hooks or browser APIs (theme toggle, motion, scroll-spy, etc.).
- Tailwind v4 with `@theme inline` in `globals.css`. Reference colors via tokens
  (`bg-background`, `text-foreground`, `text-accent`, `border-border`).
- All durable colors live as CSS variables in `:root` and `.dark`. Do not
  hard-code hex values inside component className strings.
- Motion stays minimal: one fade-up via `MotionSection`. No parallax, no scroll
  snap, no stagger trees.
- Reduced-motion preference is respected globally in `globals.css`.

## Privacy rule (hard)

The site MUST NOT contain: email, phone, postal address, location, or GitHub
links. The portrait, ORCID, Google Scholar, and LinkedIn are the only personal
artifacts surfaced. If a future change suggests adding any of the forbidden
items, stop and ask the owner first.

## Development

```bash
npm install
npm run dev          # http://localhost:3000
npm run build        # produces .next/
npm run start        # serves the production build on port 3000
npm run pdf          # regenerates public/assets/cv.pdf via Chrome headless
```

## Deploy

Vercel from this directory. `vercel.json` is checked in. No env vars, no
secrets, no server routes.

- CLI: `npm i -g vercel && vercel` then `vercel --prod`
- Or: push to GitHub, import in the Vercel dashboard
